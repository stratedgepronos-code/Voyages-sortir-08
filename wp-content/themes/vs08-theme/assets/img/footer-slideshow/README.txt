  DIAPORAMA FOOTER — DOSSIER DES PHOTOS
═══════════════════════════════════════════════════════════════

Chemin exact (sur le serveur ou en local) :
  wp-content/themes/vs08-theme/assets/img/footer-slideshow/

Fichiers utilisés : 01.jpg, 02.jpg, … 12.jpg (ou .png / .webp).
Si ce dossier est vide, le thème charge des images Pexels par défaut.

───────────────────────────────────────────────────────────────
  TAILLE RECOMMANDÉE (éviter la pixelisation au plein écran)
───────────────────────────────────────────────────────────────
  • Minimum : 2560 × 1440 px (format paysage)
  • Idéal 4K : 3840 × 2160 px
  • Format : JPG qualité 85–90 % ou WebP

───────────────────────────────────────────────────────────────
  RETIRER UNE CAPTURE D’ÉCRAN (screen) QUI TOURNE ENCORE
───────────────────────────────────────────────────────────────
  1. Ouvrez ce dossier sur le serveur (FTP / gestionnaire de fichiers).
  2. Supprimez le fichier qui est la capture (ex. 01.png ou 01.jpg).
  3. OU ajoutez son nom dans exclude.txt (une ligne = un nom de fichier).
  Les noms contenant "screen", "screenshot", "capture", "image-123…"
  sont ignorés automatiquement.

Option WordPress : vs08_footer_bg_slides = URL1|URL2|… (remplace ce dossier).
